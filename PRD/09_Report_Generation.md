# Report Generation

## Overview

Generates `NIGHTYTIDY-REPORT.md` — the single artifact the user reads the morning after a run. Contains the narrated changelog (plain-English summary), step-by-step results, and rollback instructions. This is the primary way users *feel* the value of NightyTidy.

## Dependencies

- `02_Logger.md` — logs report generation
- `04_Git_Operations.md` — commits the report as the final commit
- `06_Prompt_Library.md` — `CHANGELOG_PROMPT` used by the executor to generate the narrated section
- `07_Step_Executor.md` — provides the step results data structure

## Module: `src/report.js`

### Exported Interface

```javascript
// Generate and write the NIGHTYTIDY-REPORT.md file.
// results: the return value from executeSteps()
// narration: the plain-English changelog text from Claude Code (may be null if generation failed)
// metadata: { projectDir, branchName, tagName, originalBranch, startTime, endTime }
export async function generateReport(results, narration, metadata)
```

### Report Structure

The report is a markdown file with this exact structure:

```markdown
# NightyTidy Report — {date}

{narrated changelog text}

---

## Run Summary

- **Date**: {YYYY-MM-DD}
- **Duration**: {Xh Ym}
- **Steps completed**: {X}/{total}
- **Steps failed**: {Y}
- **Branch**: {nightytidy/run-YYYY-MM-DD-HHmm}
- **Safety tag**: {nightytidy-before-YYYY-MM-DD-HHmm}

## Step Results

| # | Step | Status | Duration | Attempts |
|---|------|--------|----------|----------|
| 1 | Documentation | ✅ Completed | 12m 34s | 1 |
| 2 | Test Coverage | ✅ Completed | 28m 07s | 1 |
| 3 | Test Hardening | ❌ Failed | 45m 12s | 4 |
| 4 | Security Audit | ✅ Completed | 19m 55s | 2 |
| ... | ... | ... | ... | ... |

## Failed Steps

### Step 3: Test Hardening
- **Error**: Claude Code exited with code 1
- **Attempts**: 4 (1 initial + 3 retries)
- **Suggestion**: Try running this step individually later with `nightytidy` and selecting only this step.

{repeat for each failed step}

## How to Undo This Run

If you need to reverse all changes from this run, ask Claude Code:

> "Reset my project to the git tag `{tagName}`"

Or run this git command:

```
git reset --hard {tagName}
```

The NightyTidy branch `{branchName}` is preserved and can be deleted manually when no longer needed.

---
*Generated by NightyTidy v{version}*
```

### Section Details

#### Narrated Changelog (Top of Report)

This is the most important section — it's the first thing the user reads. The text comes from Claude Code's response to the `CHANGELOG_PROMPT` (executed by the executor at the end of the run).

**If narration is available**: Place it directly after the title with no header. It should read as a friendly note, not a technical section.

**If narration failed** (Claude Code didn't return usable output): Replace with a fallback paragraph:

```
NightyTidy ran {X} improvement steps on your codebase. {Y} steps completed successfully.
See the step results below for details on what changed. The detailed changelog couldn't
be generated this time — check nightytidy-run.log for more information.
```

The narration is the soul of the report. If it fails, the report still works but loses its emotional impact.

#### Run Summary

Factual metadata. Durations formatted as human-readable strings:
- Under 1 hour: `Xm Ys` (e.g., "47m 23s")
- Over 1 hour: `Xh Ym` (e.g., "6h 42m")
- Over 12 hours: `Xh Ym` (e.g., "14h 03m")

#### Step Results Table

One row per selected step. Status indicators:
- `✅ Completed` — green checkmark
- `❌ Failed` — red X

Duration per step formatted the same way as total duration. Attempts column shows `1` for first-try success, `2-4` for steps that needed retries.

Steps the user deselected in the step selector are NOT included in this table — only the steps that were actually attempted.

#### Failed Steps Detail

Only present if at least one step failed. Provides enough detail for the user to understand what happened without being overwhelming. The suggestion to retry individually is a practical next step for the user.

#### Undo Instructions

Always present. Provides both the plain-English method (ask Claude Code) and the git command. The plain-English method is listed first because the target audience is vibe coders who may not be comfortable with git commands.

### CLAUDE.md Update

After writing the report, NightyTidy also appends (or updates) a section in the project's `CLAUDE.md` file with the undo instruction:

```markdown
## NightyTidy — Last Run

Last run: {date}. To undo, reset to git tag `{tagName}`.
```

If `CLAUDE.md` doesn't exist, create it with just this section. If it exists, find and replace the `## NightyTidy — Last Run` section, or append it if not found.

This ensures that the next time anyone uses Claude Code on this project, the AI knows about the NightyTidy run and how to undo it if asked.

### File Writing

- **Path**: `{projectDir}/NIGHTYTIDY-REPORT.md`
- **Mode**: Overwrite (each run produces a fresh report)
- **Encoding**: UTF-8
- **After writing**: The report file should be committed as the last commit on the run branch before the merge. The executor or finalizer handles this commit.

### Report Generation Approach

The report is built using plain string concatenation/template literals. No templating engine.

```javascript
function buildReport(results, narration, metadata) {
  let report = `# NightyTidy Report — ${metadata.date}\n\n`;
  
  if (narration) {
    report += `${narration}\n\n---\n\n`;
  } else {
    report += `${fallbackNarration(results)}\n\n---\n\n`;
  }
  
  report += buildSummarySection(results, metadata);
  report += buildStepTable(results);
  
  if (results.failedCount > 0) {
    report += buildFailedSection(results);
  }
  
  report += buildUndoSection(metadata);
  
  return report;
}
```

### Duration Formatting Helper

```javascript
function formatDuration(ms) {
  const seconds = Math.floor(ms / 1000);
  const minutes = Math.floor(seconds / 60);
  const hours = Math.floor(minutes / 60);
  
  if (hours > 0) {
    return `${hours}h ${String(minutes % 60).padStart(2, '0')}m`;
  }
  return `${minutes}m ${String(seconds % 60).padStart(2, '0')}s`;
}
```

## Testing Notes

- Test report generation with various result sets:
  - All steps completed
  - Mix of completed and failed
  - All steps failed
  - Single step run
- Test with and without narration text
- Test duration formatting: <1h, exactly 1h, >1h, >12h

- Verify the report is valid markdown (no broken tables, no unclosed formatting)

## Gaps & Assumptions

- **Report overwrite** — Each run overwrites the previous report. If users want to preserve reports across runs, that's a post-MVP feature (run history/archiving).
- **Report in git history** — The report is committed to the run branch. After merge, it's in the main branch too. Previous run reports are overwritten in the working tree but preserved in git history.
- **Narration quality** — Entirely dependent on Claude Code's response to the changelog prompt. If the response is too technical, too generic, or misses key changes, the report's emotional impact suffers. The prompt in `06_Prompt_Library.md` is carefully crafted but may need iteration.
- **Before/after diff stats** — Listed as a P2 post-MVP feature. The MVP report doesn't include lines changed, files modified, etc. The narrated changelog should reference specific numbers, but these come from Claude Code's analysis, not NightyTidy's own measurement.
